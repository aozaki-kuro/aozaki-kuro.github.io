Brief Guide of Stream Archiving

Introduction

Due the increase of Sui-chan's out-of-youtube live, youtube-dl only doesn't meet the need of archiving. Therefore I introduce my own ways to archive the streams.

Methods

1. Youtube-dl

>> Github

The famous youtube-dl is simply the best way to archive streams after the stream's over.

If you don't know how to use it, follow these steps.

For Windows Enviroment, first add a %PATH% directory[1], also remeber to install aria2c, ffmpeg, atomicparsely, beautifulsoup4 for advanced features.

The config file should be at %APPDATA%\youtube-dl\config.txt or C:\Users\<user name>\youtube-dl.conf

After configuring, open Powershell or cmd or Windows Terminal or whatever you want and type the following

1

youtube-dl 'stream-address'

2. Streamlink

>> Github

Since youtube-dl can only download the archive only after the stream ended, it won't work for these none-archive streams.

I suggest streamlink. It's also pretty easy to use.

After installing streamlink and Python, just type:

1

streamlink 'stream-address' best -o 'outputfile'

The 'outputfile' could be replaced by a path like D:\videos\20201124.mp4

Streamlink claimed to be also able to record both nicolive and nicolive timeshift, but I'm not sure if the recording will interfere with watching.

If you want to record nicolive, you will need addtional settings:

1

streamlink 'stream-address' best -o 'outputfile' --niconico-email 'EMAIL' --niconico-password 'PASSWORD'

3. Stream Recorder

>> Chrome Store

This is a plugin which can directly grab the stream. It did a great job in the SUISEI "POWER" LIVE. I successfully saved the live-record edition using this plugin.

The details were written in the plugin, read before use.

But it might be a little bit unstable. You might have to refresh and grab for several times until you see the higher resolution.

And it does not work with YouTube, so you still need streamlink.

Conclusion

These three methods were running in Windows Enviroment and worked fluently for me, as long as the network speed doesn't bottleneck.

If having any questions, read the documents on Github first.
